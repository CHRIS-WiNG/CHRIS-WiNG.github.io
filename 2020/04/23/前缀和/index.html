什么是前缀和

现有一数组 $A$ ，如果我们需要对 $A$ 查询 $m$ 次，每次给出 $L$ 、 $R$ 两个数，要求返回区间 $[L, R]$ 中元素的和，我们应该怎么做，如果每次都是遍历一次它给出的区间计算出答案，那么其时间复杂度就会达到 $O(m*n)$ ，而如果我们使用前缀和的方法来做的话就能够将时间复杂度降到 $O(n)$ ：用法如下：

顾名思义，前缀和就是前面i个数的总和，建立好前缀和数组后，对于每次询问，我们只需要计算 $prefix[R] - prefix[L - 1]$就能得到想要的答案了。

例题

560. 和为K的子数组

Difficulty: 中等

给定一个整数数组和一个整数 k，你需要找到该数组中和为 *k *的连续的子数组的个数。

示例 1 :

说明 :

  1. 数组的长度为 [1, 20,000]。
  2. 数组中元素的范围是 [-1000, 1000] ，且整数 *k *的范围是 [-1e7, 1e7]。

Thought

1.首先可以使用暴力法做，需要两层循环，第一层循环锚定一个起始元素，在此元素的基础上开始第二层循环，遍历后面所有元素，并将每个元素的值相加，如果和为 $k$ ，则结果+1，直到锚定完所有元素。
2.如果我们使用前缀和，建立一个数组 $prefix[]$ ，prefix中存放每个不同的和出现的次数，也就是： $prefix[sum] = sum\_count$ ，我们只需要一次for循环，就可以得到所有和为 $k$ 的子数组的个数：计算 $0..n$ 中每一个位置的 $sum$ ，查找 $sum - k$ 是否存在，如果存在，把存在的个数加入结果中，并把当前 $sum$ 在 $prefix$ 中的值 $prefix[sum]++$ 。（因为和的大小不可控，所以实现的时候使用HashMap）

Solution

时间复杂度： $O(n)$
空间复杂度： $O(n)$ （最坏情况下每个sum都只出现一次）

Language: Java

提示：

  • 1 <= nums.length <= 50000
  • 1 <= nums[i] <= 10^5
  • 1 <= k <= nums.length

Thought

1.可以使用双指针（滑动窗口）来做，先让右指针找到k个奇数，然后向右滑动直到遇到一个奇数停下，期间走过的偶数就是该k个奇数的子数组的可选右边界，同理，向右滑动左指针，直到遇到一个奇数停下，期间走过的偶数就是该k个奇数的子数组的可选左边界，这里切莫不要忘了两边的两个奇数也可以作为边界，将左边界数目与右边界数目相乘即可得到该子数组所有可能的组合方法。该方法时间复杂度 $O(n)$ ，空间复杂度 $O(1)$。
2.同样可以使用前缀和方法，我们这次要统计的是奇数个数 $sum$ ，这里奇数的个数是可控的，最多就是n个嘛，所以开一个n + 1的数组就可以。

Solution

时间复杂度：$O(n)$ （实测会比滑动窗口要快几倍）
空间复杂度：$O(n)$

Language: Java

总结

前缀和的思想就是空间换时间思想，一般求一个确定的目标的子数组数目这类题型适合使用前缀和思想，在使用前缀和思想时，最重要的就是要想清楚前缀和数组中存放的东西代表什么含义，数组下标代表什么含义，以及数组初始值。想清楚这些，问题也就解决一半了。
